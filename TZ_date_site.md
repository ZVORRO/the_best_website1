# ТЗ для Claude Code: сайт-приглашение на свидание

## 0. Контекст

Разовый персональный проект. Я (Кирилл) приглашаю девушку (Анастасию) на свидание через стильный сайт-«гугл-форму». Сайт — шутка с серьёзным дизайном: чёрно-белый минимализм, тонкие витиеватые узоры-текстуры, аккуратные анимации. Анастасия — начитанный проджект-менеджер, обожает Мураками и научную фантастику. Тон сайта — ироничный, использует PM-терминологию (бэклог, спринт, груминг, приоритизация, эпик, дедлайн, аутпут) в шутливой форме. Цель — вытащить её из хандры на встречу.

Бюджет — 0. Всё работает на бесплатных тарифах.

Проект уже создан локально в папке `nastya`. Внутри есть папка `photo` со всеми афишами.

## 1. Технический стек

- **Фреймворк:** Next.js (App Router) — нужен ради одной serverless-функции для безопасной отправки email через Resend (ключ нельзя светить в браузере). Next + Vercel дают это бесплатно из коробки.
- **Стилизация:** Tailwind CSS.
- **Анимации:** Framer Motion (motion). Уважать `prefers-reduced-motion`.
- **Деплой:** Vercel (бесплатный тариф).
- **Email:** Resend (бесплатный тариф: 100 писем/день). Ключ ТОЛЬКО в env `RESEND_API_KEY`, используется в serverless-функции `app/api/submit/route.ts`, НИКОГДА не в клиентском бандле.
- **Без БД.** Ответ сразу уходит мне на почту.

## 3. Нарратив и секции

Сайт скроллится как маленькая история. Каждая секция появляется по мере прокрутки (fade + slide вверх, Framer Motion `whileInView`).

1. **Hero** — на весь экран, почти чистый фон, узор по краям. Заголовок-интрига, напр.: «Кому: Анастасії. Тема: пропозиція, яку складно проіґнорувати». Кнопка-стрелка «Розгорнути порядок денний» (скролл вниз).
2. **Intro** — ироничное PM-обращение. Смысл: «У нас накопився бэклог нереалізованих вихідних. Пропоную провести грумінг і обрати, що візьмемо в спринт цього тижня». Подмигнуть Мураками/фантастике одной фразой. (Точные тексты я доработаю позже — оставь их легко редактируемыми.)
3. **OptionsGrid** — заголовок «Бэклог. Обери одну або кілька ініціатив». Сетка карточек (раздел 4) + поле «Свій варіант (тобі видніше, ти ж тут PM)».
4. **FinalQuestion** — крупный вопрос: «Анастасіє, ви готові піти на побачення з Кирилом?». Кнопки «Так» (рабочая) и «Ні» (раздел 6).
5. **SuccessScreen** — только после «Так» и успешной отправки. Тёплый текст, напр. «Епік підтверджено. Деталі — в особистих». Аккуратная финальная анимация (проявляющийся узор / монохромное конфетти).

## 4. Карточки вариантов и логика выбора

Каждая карточка: изображение (соотношение 4:5, `object-fit: cover`), название, короткое ироничное описание, состояние «выбрано» (рамка/инверсия). **Множественный выбор разрешён.**

КЛЮЧЕВОЕ: у многих вариантов есть выбор дня / времени / локации / под-варианта (фильм, спектакль, концерт). Реализуй компонент `SchedulePicker`, который раскрывается ВНУТРИ карточки, когда вариант выбран. Если у варианта несколько под-вариантов (например два фильма или три концерта) — сначала выбор под-варианта, затем доступные для него день/время.

**Глобальное правило по датам:** пятница 5 июня недоступна НИГДЕ. Исключай её из всех списков дат.

Все данные вариантов держи в `lib/options.ts` как единый массив структур, чтобы я мог править текст и расписание в одном месте.

## 6. Кнопка «Ні»

- **Desktop** (`(hover: hover) and (pointer: fine)`): при наведении «Ні» убегает в случайную точку в пределах вьюпорта (плавный transition). Не вылетает за края, не перекрывает «Так».
- **Mobile / touch:** НЕ бегает. При каждом тапе меняет текст по цепочке: «Ні» → «ну ніии» → «точно ні?» → «подумай ще» → «...ну добре», с каждым тапом уменьшается и бледнеет, в конце исчезает/неактивна.
- Кнопка «Так» всегда полностью рабочая, по клику → отправка ответа → SuccessScreen.

## 7. Дизайн-гайд

- **Палитра:** только монохром — чёрный (#0a0a0a), белый (#fafafa), 2–3 серых. Никаких цветных акцентов.
- **Узоры:** тонкий line-art (гравюра/тушь), очень светлый, по краям секций / за карточками. Inline SVG в `Ornament.tsx`. Минимализм важнее узоров — если сомневаешься, делай тише.
- **Типографика:** контрастная пара через next/font — крупный гротеск для заголовков (Unbounded или Space Grotesk) + спокойный текстовый шрифт. ОБЯЗАТЕЛЬНА поддержка кириллицы (украинский).
- **Анимации:** сдержанные, плавные (ease-out, 300–600ms). Входы по `whileInView`, hover-микро на карточках, мягкий переход на Success. Уважать `prefers-reduced-motion`.
- **Адаптив:** mobile-first (откроет с телефона).

## 9. Обратная связь (Resend) — критичная часть

1. Клиент собирает: выбранные варианты + для каждого выбранные под-вариант/день/время/локацию, текст «свій варіант», факт «Так».
2. POST на `/api/submit`.
3. `route.ts` (server-side): читает `RESEND_API_KEY`, `TO_EMAIL`, `FROM_EMAIL`; формирует читаемое письмо (тема «Анастасія відповіла 🎉», тело — список выборов с деталями + таймстамп по Києву); шлёт через Resend; возвращает `{ok:true}` или ошибку.
4. Клиент: лоадер → при успехе SuccessScreen; при ошибке мягкое «щось пішло не так, напиши Кирилу напряму» + «спробувати ще».

**Дублирование (надёжность):** если в env заданы `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID`, та же функция дублирует ответ в Telegram. Нет переменных — молча пропустить. Email основной, Telegram бонус.

`.env.example`:
```
RESEND_API_KEY=
TO_EMAIL=
FROM_EMAIL=
TELEGRAM_BOT_TOKEN=
TELEGRAM_CHAT_ID=
```
